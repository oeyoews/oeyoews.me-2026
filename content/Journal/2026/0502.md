# 0502

今天在团队讨论里，大家都提到一个共同变化：沟通内容变了。

以前 code review 常见评论是“这段可以再简化”“命名不够清晰”。现在更多是“这个 prompt 的上下文是否完整”“这个生成结果有没有隐藏假设”。AI 进入开发流程后，程序员之间的协作对象从“彼此的代码”扩展到了“彼此的思考过程”。

这其实是 AI 带来的第四阶段影响：协作粒度上移。

我们不再只评审结果，还要评审问题定义、约束条件和验证方式。谁能把需求翻译成清晰、可执行、可验证的上下文，谁就能让团队整体产出更稳定。

我越来越确定：未来优秀程序员的标志，不只是写出漂亮代码，而是能把复杂问题组织成一套可被人和 AI 共同执行的流程。

## 团队协作任务板

- [x] 在评审里补充“问题定义”检查项
- [x] 统一 prompt 基础结构（目标/约束/验收）
- [ ] 给常见任务沉淀可复用模板
- [ ] 统计一周内 AI 生成代码的返工率

> 不是“谁写得快”赢，而是“谁让团队稳定变快”赢。
